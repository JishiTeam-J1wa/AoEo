FLOW 3 ChatCompleteDual & Audit -- 双 Provider 并发对比
Dual Completion
并发请求 2 个 Provider (goroutine + WaitGroup)。
返回 DualResult { Primary, Secondary, Consensus }。
Consensus = normalize(primary.Content) == normalize(secondary.Content)
scheduler.go: ChatCompleteDual() · 并发 + MergeChoices()
Audit Mode
要求 2+ Providers。Primary + Secondary 分别完成。
共识: 合并结果; 分歧: 并列展示 + Emit(audit:disagree)。
返回 AuditResult { Primary, Audit, Consensus, Adjusted }
audit.go: Audit() · result.go: Consensus() + MergeChoices()
FLOW 4 ChatCompleteStream -- SSE 流式链路
S1
Semaphore.Acquire → Provider.ChatCompleteStream
获取信号量，调用 Provider 的流式接口。Provider 返回 <-chan StreamCompletionResponse。
S2
Buffered Wrapper Channel (16)
外层创建 16-buffer channel，解耦生产者 (Provider goroutine) 和消费者 (caller)。Producer goroutine 读取内部 channel → ApplyAfterStreamChunk → 写入外层 channel。
S3
Per-Chunk Privacy Restore PII
每个 chunk 的 Delta.Content 通过 restoreStreamChunk: 查找 session 映射 → fake → original 替换。流完成后调用 AfterStreamDone。
pseudonymizer.go: RestoreStreamChunk() → 逐 chunk 替换
S4
Semaphore.Release → History.Record
消费完毕后释放信号量，记录调用历史。
DETAIL Privacy Pseudonymize / Restore 内部流程
PseudonymizeRequest BEFORE
1. 收集所有 messages 的 content
2. DetectBatch(texts) → OPF 批量检测
3. 按 Span 长度 降序排序
4. 遍历 Spans:
  a. GetFake(session, original) → 已有映射?
  b. 无 → FakeGenerator.Generate(type, text)
  c. 碰撞检测: fake 不与其他 original 重叠
  d. Pebble.Set(session, fake, original, type)
  e. strings.Replace 替换 content
5. 返回修改后的 request + mappings
RestoreResponse AFTER
1. GetSession(sessionID) → 获取所有映射
2. 遍历 response choices:
  a. 遍历 mappings (长 fake 优先)
  b. replaceFake(content, fake, original)
  c. 模糊匹配: 处理标点边界
3. detectLeaks(resp, entries):
  a. 扫描 content 中残留的 fake 值
  b. 有泄露 → 日志 Warn
4. 返回还原后的 response
RESILIENCE Provider 熔断器 & 健康监控状态机
CLOSED HEALTHY
正常服务中
failCount < MaxFailures
sliding window 记录延迟/成功率
failCount ≥ 3 →
OPEN TRIPPED

failUntil = now + CooldownDuration
IsAvailable() = false
