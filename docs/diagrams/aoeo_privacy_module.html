PIPELINE 四阶段隐私处理流水线
STAGE 1
Detect
OPF /redact API 检测 PII。
Batch 模式: 所有 messages 一次提交。
30+ OPF labels → 10 EntityTypes 归一化。
timeout: 15s (single) / 30s (batch)
STAGE 2
Pseudonymize
按 Span 长度降序排序，避免短串干扰。
复用已有映射（同一 session）。
碰撞检测: 确保 fake 不与已有值冲突。
Pebble KV 原子双向写入
STAGE 3
Forward
脱敏后的请求发送给 AI Provider。
Provider 只看到 fake 数据。
原始 PII 不离开网关。
Interceptor BeforeRequest hook
STAGE 4
Restore
响应中 fake → original 还原。
模糊匹配: 处理 AI 添加的标点边界。
泄露扫描: 还原后检测残留 fake。
Streaming: chunk 级实时还原

POST /redact/batch → opfBatchRedactResponse

重试: 2 次, 指数退避 (100ms, 200ms)
